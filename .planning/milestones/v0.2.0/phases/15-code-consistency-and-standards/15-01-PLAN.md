    **CON-01: Structured logger replacement**

    1. First, identify all service files missing a logger declaration:

       ```python
       import logging

       logger = logging.getLogger(__name__)
       ```

    3. Then find and replace all `print()` calls in `backend/app/services/`:
       ```bash
       grep -rn 'print(' backend/app/services/ --include='*.py' | grep -v '__pycache__'
       ```

    4. For each `print()` call, convert to the appropriate log level:
       - Progress/status messages -> `logger.info()`
       - Warning conditions -> `logger.warning()`
       - Error messages -> `logger.error()`
       - Debug/diagnostic output -> `logger.debug()`

       Use `%s` format strings (not f-strings) for logger calls per Python logging best practices:
       ```python
       # Before:
       print(f"Processing trigger {trigger_id}")
       # After:
       logger.info("Processing trigger %s", trigger_id)
       ```

    5. Do NOT touch print() calls in:
       - `backend/app/db/migrations.py` (migration progress -- may run before logging is configured; per Pitfall 4 from research)

       - Test files
       - CLI scripts outside app/

    6. After replacement, verify zero print() calls remain in services:

       ```bash
       grep -rn 'print(' backend/app/services/ --include='*.py' | grep -v '__pycache__'
       ```
