  <files>frontend/src/services/api/bot-templates.ts, frontend/src/services/api/prompt-snippets.ts, frontend/src/services/api/triggers.ts, frontend/src/services/api/types.ts, frontend/src/services/api/index.ts</files>
  <action>
    Following the existing API client pattern (e.g., frontend/src/services/api/triggers.ts, frontend/src/services/api/marketplace.ts):

    1. Add TypeScript types to `frontend/src/services/api/types.ts`:
       - `BotTemplate`: id, slug, name, description, category, icon, config_json (string), sort_order, source, is_published, created_at
       - `BotTemplateDeployResponse`: trigger_id, template_id, trigger_name, message
       - `PromptSnippet`: id, name, content, description, is_global, created_at, updated_at
       - `CreateSnippetRequest`: name, content, description?
       - `UpdateSnippetRequest`: name?, content?, description?
       - `PromptHistoryEntry`: id, trigger_id, old_template, new_template, author, diff_text, changed_at
       - `PreviewPromptFullResponse`: rendered_prompt, cli_command, cli_command_parts (string[]), backend_type, model, trigger_name, unresolved_placeholders (string[]), unresolved_snippets (string[])

    2. Create `frontend/src/services/api/bot-templates.ts`:
       - `export const botTemplateApi = {`
         - `list: () => apiFetch<{ templates: BotTemplate[] }>('/admin/bot-templates')`
         - `get: (id: string) => apiFetch<BotTemplate>('/admin/bot-templates/' + id)`
         - `deploy: (id: string) => apiFetch<BotTemplateDeployResponse>('/admin/bot-templates/' + id + '/deploy', { method: 'POST' })`
       - `}`

    3. Create `frontend/src/services/api/prompt-snippets.ts`:
       - `export const promptSnippetApi = {`
         - `list: () => apiFetch<{ snippets: PromptSnippet[] }>('/admin/prompt-snippets')`
         - `get: (id: string) => apiFetch<PromptSnippet>('/admin/prompt-snippets/' + id)`
         - `create: (data: CreateSnippetRequest) => apiFetch<PromptSnippet>('/admin/prompt-snippets', { method: 'POST', body: JSON.stringify(data) })`
         - `update: (id: string, data: UpdateSnippetRequest) => apiFetch<PromptSnippet>('/admin/prompt-snippets/' + id, { method: 'PUT', body: JSON.stringify(data) })`
         - `delete: (id: string) => apiFetch<void>('/admin/prompt-snippets/' + id, { method: 'DELETE' })`
         - `resolve: (text: string) => apiFetch<{ resolved: string }>('/admin/prompt-snippets/resolve', { method: 'POST', body: JSON.stringify({ text }) })`
       - `}`

    4. Add new methods to existing `triggerApi` in `frontend/src/services/api/triggers.ts`:
       - `getPromptHistory: (triggerId: string) => apiFetch<{ history: PromptHistoryEntry[] }>('/admin/triggers/' + triggerId + '/prompt-history')`
       - `rollbackPrompt: (triggerId: string, versionId: number) => apiFetch<any>('/admin/triggers/' + triggerId + '/rollback-prompt', { method: 'POST', body: JSON.stringify({ version_id: versionId }) })`
       - `previewPromptFull: (triggerId: string, payload: Record<string, any>) => apiFetch<PreviewPromptFullResponse>('/admin/triggers/' + triggerId + '/preview-prompt-full', { method: 'POST', body: JSON.stringify(payload) })`
       - Note: The NL generate stream endpoint (`/admin/triggers/generate/stream`) will use EventSource directly in the component, not apiFetch

    5. Re-export new API objects from `frontend/src/services/api/index.ts`:
       - Add `export { botTemplateApi } from './bot-templates'`
       - Add `export { promptSnippetApi } from './prompt-snippets'`
       - Add type exports for BotTemplate, PromptSnippet, etc.
  </action>
