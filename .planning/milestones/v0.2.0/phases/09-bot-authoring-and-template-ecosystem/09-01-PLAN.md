    Following the exact pattern of PluginGenerationService, CommandGenerationService, and other BaseGenerationService subclasses (09-RESEARCH.md Recommendation 4):

    1. Create `backend/app/services/trigger_generation_service.py`:
       - `class TriggerGenerationService(BaseGenerationService):`
       - `_gather_context(cls) -> dict`: Return dict with:
         - `existing_triggers`: list of existing trigger names/types for context (call `get_all_triggers()`)
         - `backends`: list of valid backend types from VALID_BACKENDS constant
         - `trigger_sources`: list of valid trigger sources (webhook, github, schedule, manual)
         - `placeholders`: dict of available placeholder variables by trigger_source (e.g. github: {pr_url}, {pr_title}, {pr_author}; webhook: {message}, {paths}; schedule: {paths})
       - `_build_prompt(cls, description: str, context: dict) -> str`: Build a prompt that:
         - Instructs Claude to generate a complete trigger configuration as JSON
         - Provides the JSON schema matching CreateTriggerRequest fields (name, prompt_template, backend_type, trigger_source, model, schedule_type, schedule_value, paths)
         - Includes context about existing triggers to avoid name conflicts
         - Lists available backends, trigger sources, and placeholder variables
         - Emphasizes generating a detailed, effective prompt_template
         - Instructs to output ONLY valid JSON, no markdown fences
       - `_validate(cls, config: dict) -> tuple`: Validate the generated config:
         - Check required fields: name, prompt_template, backend_type, trigger_source
         - Check backend_type is in VALID_BACKENDS
         - Check trigger_source is valid
         - If schedule trigger, check schedule_type and schedule_value present
         - Return (config, warnings_list) — add warnings for missing optional fields, do NOT reject for missing optional fields

    2. Add NL generation endpoint to `backend/app/routes/triggers.py`:
       - `@triggers_bp.post("/generate/stream")` — following the exact pattern of commands.py line 114-131

       - Return `Response(TriggerGenerationService.generate_streaming(description), mimetype="text/event-stream", headers={"Cache-Control": "no-cache", "X-Accel-Buffering": "no"})`

    - Do NOT reimplement SSE streaming — BaseGenerationService.generate_streaming() handles all subprocess management, threading, JSON parsing, and SSE protocol
