<task type="auto">
  <name>Task 2: Enhance workflow DAG validation with descriptive error messages</name>
  <files>backend/app/routes/workflows.py, backend/app/services/workflow_execution_service.py, backend/tests/test_dag_validation.py</files>
  <action>
    1. In `backend/app/services/workflow_execution_service.py` (or a new `backend/app/services/workflow_validation_service.py` if the existing file is too large):
       - Create or enhance a `validate_workflow_dag(graph_data: dict) -> Tuple[bool, List[str]]` function that returns (is_valid, error_messages)
       - The function should check for:

       a. **Cycle detection**: Use `graphlib.TopologicalSorter` to detect cycles. If `CycleError` is raised, extract the cycle nodes and return a descriptive error:
          `"Cycle detected: node-A -> node-B -> node-C -> node-A"`

       b. **Missing node references**: For each edge in the graph, verify both source and target node IDs exist in the nodes list. If a reference is missing:
          `"Edge references missing node 'node-xyz': no node with this ID exists in the workflow"`

       c. **Invalid condition expressions**: For conditional/branch nodes, validate the condition expression is syntactically valid. Use `ast.parse()` (from Python stdlib) to check syntax. If the expression is unparseable:
          `"Invalid condition expression in node 'condition-1': SyntaxError at column 5"`
          Only allow safe expression syntax (comparisons, boolean operators, string operations). Reject expressions containing import, exec, eval, __builtins__, or other dangerous constructs.

       d. **Unreachable nodes**: Identify nodes with no incoming edges and no outgoing edges (isolated nodes). Warn about these but don't reject.

    2. In `backend/app/routes/workflows.py`:
       - In the workflow creation endpoint (POST), call `validate_workflow_dag(graph_data)` before saving
       - If validation fails, return 400 with the list of error messages
       - In the workflow update endpoint (PUT), also call validation before saving
       - Add a standalone validation endpoint:
         ```python
         @workflows_bp.post("/validate")
         def validate_workflow():
             """Validate a workflow DAG without saving (API-09)."""
             data = request.get_json()
             graph = data.get("graph", {})
             is_valid, errors = validate_workflow_dag(graph)
             return {"valid": is_valid, "errors": errors}, HTTPStatus.OK if is_valid else HTTPStatus.BAD_REQUEST
         ```

    3. Write `backend/tests/test_dag_validation.py`:
       - Test valid DAG: linear A->B->C passes validation
       - Test cycle detection: A->B->C->A returns error with cycle description
       - Test missing node reference: edge referencing non-existent node returns error identifying the missing node
       - Test invalid condition expression: node with `condition: "if x ==== y"` returns syntax error
       - Test dangerous expression rejection: node with `condition: "__import__('os').system('rm -rf /')"` is rejected
       - Test valid condition expression: `condition: "output.status == 'success'"` passes
       - Test standalone validation endpoint returns valid=True for valid DAG
       - Test standalone validation endpoint returns valid=False with errors for invalid DAG
       - Test that workflow creation with cycle is rejected (400 status)
       - Use `isolated_db` fixture

    AVOID: Do NOT use eval() or exec() to validate expressions -- use ast.parse() for syntax checking only (per Phase 7 decision: AST-based expression evaluator).
    AVOID: Do NOT break existing workflow creation/update functionality -- validation is additive.
