<objective>
Implement the integration adapter framework with Slack, Teams, JIRA, and Linear adapters, a unified notification/ticket service, and an inbound Slack slash command handler for triggering bot executions from Slack.

Purpose: This plan delivers INT-01 (Slack/Teams bidirectional integration including inbound slash commands) and INT-02 (JIRA/Linear auto-issue creation). Adapters use the secrets vault (Plan 02) for credentials and audit trail (Plan 01) for logging.

Output: Pluggable adapter framework, 4 adapters (Slack, Teams, JIRA, Linear), notification dispatch service, Slack slash command inbound endpoint, integration configuration CRUD, comprehensive tests with mocked external APIs.

Research basis: Adapter pattern (Gamma et al. 1994), slack-sdk v3.x, Slack slash command verification (api.slack.com/authentication/verifying-requests-from-slack), jira library, Teams Incoming Webhooks, Linear GraphQL API. See 11-RESEARCH.md for code examples and anti-patterns.
</objective>

       CREATE TABLE IF NOT EXISTS integrations (
           id TEXT PRIMARY KEY,
           name TEXT NOT NULL,
           type TEXT NOT NULL,

       ```python
       from abc import ABC, abstractmethod
       from typing import Dict, Optional, Tuple

       class IntegrationAdapter(ABC):
           @abstractmethod
           def send_notification(self, channel: str, message: str, metadata: dict = None) -> bool: ...

           @abstractmethod
           def create_ticket(self, project_key: str, finding: dict) -> Optional[str]: ...

           @abstractmethod
           def validate_config(self, config: dict) -> Tuple[bool, Optional[str]]: ...

       ADAPTER_REGISTRY: Dict[str, type] = {}

       def register_adapter(adapter_type: str, adapter_class: type):
           ADAPTER_REGISTRY[adapter_type] = adapter_class

       def get_adapter(adapter_type: str, **kwargs) -> Optional[IntegrationAdapter]:
           cls = ADAPTER_REGISTRY.get(adapter_type)
           if cls:
               return cls(**kwargs)
           return None
       ```

    1. **Slack adapter** -- `backend/app/services/integrations/slack_adapter.py`:
       - Constructor takes `token: str` (retrieved from secrets vault by caller)
       - `send_notification()`: Use `slack_sdk.WebClient.chat_postMessage()` with Block Kit formatting (see research example). Format execution summary with status emoji, bot name, duration.
       - `create_ticket()`: Return None (Slack doesn't create tickets)
       - `validate_config()`: Check token is non-empty, optionally call `auth.test` API to verify scopes
       - Register via `register_adapter("slack", SlackAdapter)` at module level
       - Handle `SlackApiError` gracefully with logging
       - Also add a classmethod `verify_slack_signature(signing_secret, timestamp, body, signature) -> bool` for slash command verification (HMAC-SHA256 per Slack docs)

    2. **Teams adapter** -- `backend/app/services/integrations/teams_adapter.py`:
       - Constructor takes `webhook_url: str`
       - `send_notification()`: POST MessageCard JSON to webhook_url via `httpx.post()` with 10s timeout (see research example)
       - `create_ticket()`: Return None (Teams doesn't create tickets)
       - `validate_config()`: Check webhook_url is non-empty and starts with https://
       - Register via `register_adapter("teams", TeamsAdapter)`

    3. **JIRA adapter** -- `backend/app/services/integrations/jira_adapter.py`:
       - Constructor takes `server: str, email: str, api_token: str`
       - `send_notification()`: Return False (JIRA doesn't support notifications)
       - `create_ticket()`: Use `jira.JIRA.create_issue()` with severity-to-priority mapping (see research example). Map: critical->Highest, high->High, medium->Medium, low->Low. Add labels ["agented", "automated"].
       - `validate_config()`: Check server, email, api_token all non-empty
       - Register via `register_adapter("jira", JiraAdapter)`

    4. **Linear adapter** -- `backend/app/services/integrations/linear_adapter.py`:
       - Constructor takes `api_key: str`
       - Use raw `httpx` with GraphQL mutations (no Python SDK per research recommendation)
       - `send_notification()`: Return False
       - `create_ticket()`: POST GraphQL mutation `issueCreate` to `https://api.linear.app/graphql` with Authorization bearer header. Include title, description, priority mapping.
       - `validate_config()`: Check api_key non-empty
       - Register via `register_adapter("linear", LinearAdapter)`

    Avoid: Per 11-RESEARCH.md, use `slack-sdk` not raw HTTP for Slack (it handles rate limiting, retry, token rotation). Do NOT make synchronous external API calls in request handlers -- all adapter calls should happen in background threads.

    3. **Slack slash command endpoint** -- Add to `backend/app/routes/integrations.py`:
       - `POST /api/integrations/slack/command` -- inbound Slack slash command webhook
         a. Verify Slack HMAC-SHA256 request signature using `SlackAdapter.verify_slack_signature()`:
            - Read `X-Slack-Signature` header and `X-Slack-Request-Timestamp` header
            - Read raw request body
            - Read Slack signing secret from SecretVaultService (stored as secret named `slack_signing_secret`)
            - Compute HMAC-SHA256 and compare with constant-time comparison (`hmac.compare_digest`)
            - If invalid, return 401
            - If timestamp is older than 5 minutes, return 401 (replay protection)
         b. Parse slash command payload: `command`, `text`, `user_id`, `channel_id`, `response_url`
         c. Parse `text` field expecting format: `run <trigger-name> on <target>` (e.g., `/agented run security-audit on payments-service`)
            - Extract trigger name and target (repo URL or project path)
            - Look up trigger by name
            - If not found, return ephemeral Slack message with error
         d. Dispatch execution via `ExecutionService` or `OrchestrationService` in a background thread
         e. Return immediate ephemeral Slack response (within 3 seconds, Slack requirement):
            ```json
            {"response_type": "ephemeral", "text": "Starting security-audit on payments-service..."}
            ```
         f. When execution completes, POST result summary to `response_url` (delayed response)
         g. Audit log the slash command invocation

       NOTE: This route uses `/api/` prefix (not `/admin/`) because it's a public webhook endpoint that Slack sends requests to.
