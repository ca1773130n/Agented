    Based on GitOps principles and existing GrdSyncService/PluginFileWatcher patterns:

    1. **DB schema** -- Add to `schema.py` and migration:
       ```sql
       CREATE TABLE IF NOT EXISTS gitops_repos (
           id TEXT PRIMARY KEY,
           name TEXT NOT NULL,
           repo_url TEXT NOT NULL,
           branch TEXT NOT NULL DEFAULT 'main',
           config_path TEXT NOT NULL DEFAULT 'agented/',
           poll_interval_seconds INTEGER DEFAULT 60,
           last_sync_at TIMESTAMP,
           last_commit_sha TEXT,
           enabled INTEGER DEFAULT 1,
           created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
       );

       CREATE TABLE IF NOT EXISTS gitops_sync_log (
           id INTEGER PRIMARY KEY AUTOINCREMENT,
           repo_id TEXT NOT NULL,
           commit_sha TEXT,
           files_changed INTEGER DEFAULT 0,
           files_applied INTEGER DEFAULT 0,
           files_conflicted INTEGER DEFAULT 0,
           status TEXT DEFAULT 'pending',
           details TEXT,
           created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
           FOREIGN KEY (repo_id) REFERENCES gitops_repos(id) ON DELETE CASCADE
       );
       CREATE INDEX IF NOT EXISTS idx_gitops_sync_repo ON gitops_sync_log(repo_id);
       ```

    2. **DB CRUD** -- Create `backend/app/db/gitops.py`:
       - `create_gitops_repo(name, repo_url, branch, config_path, poll_interval) -> str`
       - `get_gitops_repo(repo_id) -> Optional[dict]`
       - `list_gitops_repos() -> list[dict]`
       - `update_gitops_repo(repo_id, **kwargs) -> bool`
       - `delete_gitops_repo(repo_id) -> bool`
       - `update_sync_state(repo_id, commit_sha, synced_at) -> bool`
       - `add_sync_log(repo_id, commit_sha, files_changed, files_applied, files_conflicted, status, details) -> int`
       - `list_sync_logs(repo_id, limit=20) -> list[dict]`
       - Re-export from `db/__init__.py`.

    3. **GitOps sync service** -- Create `backend/app/services/gitops_sync_service.py`:
       - `GitOpsSyncService` class with classmethods:

       - `sync_repo(repo_id, dry_run=False) -> dict`:
         a. Load repo config from DB
         b. Clone or pull the git repo to a temp directory using `subprocess.run(["git", "clone", ...])` or `subprocess.run(["git", "pull"])` if already cloned. Use a persistent local cache dir under `backend/.gitops-cache/{repo_id}/`.
         c. Check current HEAD commit SHA. If same as `last_commit_sha`, skip (no changes).
         d. List YAML files in `config_path` directory: `glob.glob(f"{local_path}/{config_path}/*.yaml")`
         e. For each YAML file, parse with `yaml.safe_load()`
         f. Compare against current DB state using ConfigExportService:
            - Export current trigger config
            - Hash both configs (use `hashlib.sha256`)
            - If hashes differ, this is a change to apply
         g. If `dry_run=True`, return list of changes without applying
         h. If `dry_run=False`:
            - Apply changes via `ConfigExportService.import_trigger(config_str, upsert=True)` -- this uses the upsert flag from Plan 11-03 to update existing triggers by name match rather than creating duplicates
            - Audit log every change with before/after state
            - If both DB and Git changed since last sync, log conflict warning and apply Git version (Git wins)
         i. Update `last_commit_sha` and `last_sync_at` in DB
         j. Add sync log entry
         k. Return summary: `{commit_sha, files_changed, files_applied, files_conflicted, changes: [...]}`

       - `start_polling()`:
         a. For each enabled gitops_repo, register an APScheduler job
         b. Job interval = `poll_interval_seconds` (default 60s)
         c. Each job calls `sync_repo(repo_id)`

       - `stop_polling()`: Remove all gitops scheduler jobs

       - `init()`: Called from `create_app()` to start polling on server start

    Avoid: Do NOT use `yaml.load()` -- always use `yaml.safe_load()` for security. Per 11-RESEARCH.md Pitfall 3, use content hashing to detect conflicts rather than timestamps. Do NOT sync if repo is private and no credentials are configured -- fail gracefully with clear error. Store the local clone cache outside the DB (filesystem). CRITICAL: Use `import_trigger(upsert=True)` not `import_trigger()` -- the upsert flag is what prevents duplicate trigger creation on subsequent syncs.
