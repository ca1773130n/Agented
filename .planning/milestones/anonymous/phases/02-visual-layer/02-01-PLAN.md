  <name>Task 2: Create TourSpotlight component and refactor TourOverlay</name>
  <files>frontend/src/components/tour/TourSpotlight.vue, frontend/src/components/tour/TourOverlay.vue, frontend/src/components/tour/__tests__/TourOverlay.test.ts, frontend/src/components/tour/__tests__/TourSpotlight.test.ts</files>
  <action>
    **Create `frontend/src/components/tour/TourSpotlight.vue`:**

    A focused component that renders the spotlight highlight with box-shadow dimming. Props:
    - `targetRect: DOMRect | null` — the bounding rect of the highlighted element
    - `visible: boolean` — controls opacity for fade transitions

    Implementation:
    - `position: fixed` container with `pointer-events: none`
    - Inline styles computed from `targetRect`: `top`, `left`, `width`, `height` (with padding from `--tour-spotlight-padding`)
    - `box-shadow: 0 0 0 9999px var(--tour-overlay-dim)` for the dimming effect
    - `border-radius: var(--tour-spotlight-radius)`
    - `z-index: var(--z-tour-spotlight)`
    - CSS transition on `top`, `left`, `width`, `height` using `var(--tour-transition-speed)` for smooth step-to-step movement
    - Inner `.tour-spotlight-glow` div for the pulsing glow effect:
      - `border: 2px solid var(--tour-glow-color)` instead of hardcoded `#818cf8`
      - Glow animation uses `var(--tour-glow-dim)` and `var(--tour-glow-bright)` instead of hardcoded `rgba(99, 102, 241, ...)`
      - Animate `opacity` on a fixed box-shadow (not animate box-shadow itself) to avoid paint thrashing per Research Pitfall 6
    - `v-if="targetRect"` — does not render when no target

    **ZERO hardcoded color or z-index values.** Every visual value must reference a CSS custom property.

    **Refactor `frontend/src/components/tour/TourOverlay.vue`:**

    TourOverlay becomes the orchestrator. It keeps:
    - All target-finding logic (findTarget, MutationObserver, scroll/resize listeners)
    - The `targetRect` ref and tracking infrastructure
    - ResizeObserver on the target element (NEW — add this for content reflow detection per Research Recommendation 2)
    - Spinner/loading state when target not found
    - Dim fallback when no target
    - Emit `next` and `skip` events

    It delegates to:
    - `<TourSpotlight :target-rect="targetRect" :visible="!!targetRect" />` for the visual highlight
    - The bottom bar stays in TourOverlay for now (will be extracted to TourProgressBar in Plan 02)

    Replace ALL hardcoded values in TourOverlay's `<style>`:
    | Old hardcoded | New CSS custom property |
    |---|---|
    | `z-index: 10000` | `z-index: var(--z-tour-overlay)` |
    | `rgba(0, 0, 0, 0.7)` | `var(--tour-overlay-dim)` |
    | `z-index: 10002` | `z-index: var(--z-tour-tooltip)` |
    | `z-index: 10003` | `z-index: var(--z-tour-controls)` |
    | `#818cf8` | `var(--tour-glow-color)` or `var(--accent-cyan)` |
    | `#1a1a2e` | `var(--bg-tertiary)` |
    | `#e4e4e7` | `var(--text-primary)` |
    | `#a1a1aa` | `var(--text-secondary)` |
    | `#71717a` | `var(--text-tertiary)` |
    | `#6366f1` | `var(--accent-cyan)` |
    | `rgba(129, 140, 248, 0.2)` | `var(--border-default)` |
    | `rgba(0, 0, 0, 0.4)` | `var(--tour-overlay-dim)` (or a secondary shadow var) |
    | `rgba(255, 255, 255, 0.1)` | `var(--border-default)` |
    | `rgba(255, 255, 255, 0.2)` | `var(--border-strong)` |
    | `white` (button text) | `var(--text-on-accent)` |

    Add ResizeObserver to the target tracking:
    - When `targetEl` is found, create a `ResizeObserver` on it that calls `updateRect()`
    - Disconnect the ResizeObserver in `cleanup()` and when target changes
    - This catches content reflow (e.g., lazy-loaded content pushing the target down)

    Change spotlight transition from `300ms` to `var(--tour-transition-speed)` (200ms).

    **Update `frontend/src/components/tour/__tests__/TourOverlay.test.ts`:**
    - Update existing tests to work with the refactored component (TourSpotlight is a child now)
    - Add test: "renders TourSpotlight child component when target is found"
    - Keep all existing behavioral tests (next/skip emits, substep label, spinner)

    - Test: does not render when targetRect is null

    - Test: all styles use CSS custom properties — grep the component source for hardcoded `#[0-9a-fA-F]`, `rgb(`, `rgba(` and assert count is 0. This is the static analysis check from the research verification strategy.
