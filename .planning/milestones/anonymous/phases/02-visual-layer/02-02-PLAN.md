    **Create `frontend/src/components/tour/TourTooltip.vue`:**

    Based on @floating-ui/vue docs (Context7, 1069 snippets) and Research Pattern 1 (Virtual Element Bridge):

    Props:
    - `targetRect: DOMRect | null` — the spotlight's tracked target rect (becomes the virtual reference)
    - `title: string` — step title
    - `message: string` — step description
    - `visible: boolean` — controls rendering and fade animation
    - `placement?: 'top' | 'bottom' | 'left' | 'right'` — preferred placement, default `'bottom'`

    Implementation:

    1. **Virtual Element Bridge** — Create a `computed` that returns a NEW virtual element object whenever `targetRect` changes. This is critical per Research Pitfall 3 (reactive virtual reference):
       ```ts
       const virtualReference = computed(() => {
         if (!props.targetRect) return null
         const r = props.targetRect
         return {
           getBoundingClientRect: () => ({
             x: r.x, y: r.y, top: r.top, left: r.left,
             right: r.right, bottom: r.bottom,
             width: r.width, height: r.height,
           }),
         }
       })
       ```

    2. **Floating UI setup** — Use `useFloating` composable:
       ```ts
       const floating = ref<HTMLElement | null>(null)
       const floatingArrow = ref<HTMLElement | null>(null)

       const { floatingStyles, placement: actualPlacement, middlewareData } = useFloating(
         virtualReference,
         floating,
         {
           placement: computed(() => props.placement ?? 'bottom'),
           middleware: [offset(12), flip(), shift({ padding: 8 }), arrow({ element: floatingArrow })],
           whileElementsMounted: autoUpdate,
         },
       )
       ```
       Import: `useFloating, offset, flip, shift, arrow, autoUpdate` from `@floating-ui/vue`.

    3. **Two-phase transition** (per Research Pitfall 1 — prevent tooltip flicker):
       - Track a local `isTransitioning` ref
       - When `visible` changes from one step to the next (targetRect changes significantly):
         - Set opacity to 0 (fade out over ~200ms)
         - After transition completes, Floating UI recomputes position
         - Set opacity to 1 (fade in at new position)
       - Use CSS `transition: opacity var(--tour-transition-speed) ease, transform var(--tour-transition-speed) ease`
       - Add slight `translateY(4px)` on hidden state for a subtle slide-in effect
       - Use `v-if="visible && targetRect"` to conditionally render

    4. **Arrow element** — Positioned via `middlewareData.arrow?.x` / `middlewareData.arrow?.y`:
       - Absolute-positioned div inside the tooltip
       - Uses CSS custom properties for color (matches tooltip background)
       - Arrow side computed from `actualPlacement` (bottom placement → arrow on top, etc.)

    5. **Styles** — ALL CSS custom properties, ZERO hardcoded values:
       - `z-index: var(--z-tour-tooltip)`
       - Background: `var(--bg-elevated)` (darker tooltip bg)
       - Border: `1px solid var(--border-default)`
       - Title text: `var(--text-primary)`, font-weight 600, font-size 14px
       - Message text: `var(--text-secondary)`, font-size 13px, line-height 1.5
       - Arrow background: matches tooltip bg `var(--bg-elevated)`
       - Arrow border: `var(--border-default)`
       - Box shadow: subtle `0 8px 32px var(--tour-overlay-dim)` for depth
       - Border-radius: `var(--tour-spotlight-radius)` (8px)
       - `pointer-events: auto` — tooltip IS interactive
       - Max-width: 320px, padding: 16px
       - Font-family: inherit (gets Geist from parent)
