    - Add `tabindex="-1"` to the `.tour-overlay` root div so it can receive focus for keydown events
    - In the script `onMounted` lifecycle, add a `keydown` event listener on `document` (not window — document captures keyboard before page content):
      ```typescript
      function handleKeydown(e: KeyboardEvent) {
        if (!props.active || !props.step) return
        if (e.key === 'Enter') {
          e.preventDefault()
          emit('next')
        } else if (e.key === 'Escape') {
          if (props.step.skippable) {
            e.preventDefault()
            emit('skip')
          }
          // Non-skippable: do nothing (OB-33: Escape does nothing when not skippable)
        }
      }
      document.addEventListener('keydown', handleKeydown)
      ```
    - Remove the listener in `onUnmounted` (already called cleanup — add keyboard cleanup there)
    - Import `onMounted` from vue (it may already be imported — check and add if missing)

    - Add test: 'Escape key does nothing when step is not skippable' — step with skippable:false, trigger keydown Escape, expect no skip emitted
    - Add test: 'keyboard handler is inactive when active=false' — set active=false, trigger Enter, expect no next emitted
    - Add test: 'overlay click does not emit dismiss or any event' — trigger click on .tour-overlay root, expect no unexpected events
