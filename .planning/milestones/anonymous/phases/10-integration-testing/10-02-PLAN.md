  <files>frontend/src/composables/__tests__/useTourMachine.test.ts</files>
  <action>
    Create `frontend/src/composables/__tests__/useTourMachine.test.ts` following Pattern 2 from 10-RESEARCH.md. This composable has module-level singleton state (`sharedActor`, `sharedInstanceId`, `subscriberCount`, `initPromise`) that MUST be reset between tests.

    **Setup pattern (CRITICAL — follow exactly):**
    ```typescript
    // Mock Vue lifecycle hooks BEFORE any other imports
    vi.mock('vue', async () => {
      const actual = await vi.importActual<typeof import('vue')>('vue');
      return { ...actual, onUnmounted: vi.fn() };
    });

    // Mock API client module
    vi.mock('../../services/api/client', () => ({
      API_BASE: '',
      getApiKey: () => null,
    }));

    beforeEach(() => {
      localStorage.clear();
      vi.resetModules();
      // Default: mock fetch to return a valid instance-id
      globalThis.fetch = vi.fn().mockResolvedValue({
        ok: true,
        json: () => Promise.resolve({ instance_id: 'test-uuid-1' }),
      }) as unknown as typeof fetch;
    });
    ```

    After each `vi.resetModules()`, import useTourMachine via `await import('../useTourMachine')`. This gives a fresh module with new singleton state.

    **Test structure with describe blocks:**

    **1. "initialization"**
    - Actor initializes in idle state: call `useTourMachine()`, wait for async init via `await vi.waitFor(() => expect(tour.state.value).not.toBeNull())`, assert `state.value === 'idle'`
    - Fetches instance-id on init: verify `fetch` was called with URL matching `/health/instance-id`
    - Handles fetch failure gracefully: mock fetch to reject, verify actor still starts in idle

    **2. "persistence — saving"**
    - After state transition, localStorage contains persisted data: start tour (send START), check `localStorage.getItem('agented-tour-machine-state')` is non-null, parse JSON and verify `schemaVersion: 1` and `instanceId: 'test-uuid-1'`
    - Persists on every transition: send START then NEXT, verify localStorage updated after each

    **3. "persistence — restoring"**
    - Restores state from valid localStorage: pre-populate localStorage with `{ schemaVersion: 1, instanceId: 'test-uuid-1', snapshot: <valid snapshot at welcome state> }`. To get a valid snapshot: create a temporary actor, send START, call `actor.getPersistedSnapshot()`, use that. Then init composable, verify state restores to 'welcome' (not idle).
    - Getting a valid snapshot for test: use a helper function that creates a temp actor from `tourMachine`, sends events to reach the desired state, gets persisted snapshot

    **4. "persistence — schema version mismatch"**
    - Pre-populate localStorage with `{ schemaVersion: 999, instanceId: 'test-uuid-1', snapshot: {...} }`, init composable, verify state is 'idle' (not restored), verify localStorage was cleared (item removed)

    **5. "persistence — instance-id mismatch"**
    - Pre-populate localStorage with `{ schemaVersion: 1, instanceId: 'old-uuid', snapshot: {...} }`, mock fetch to return `{ instance_id: 'new-uuid' }`, init composable, verify state is 'idle' (discarded stale state), verify localStorage was cleared

    **6. "persistence — instance-id match (valid restore)"**
    - Pre-populate localStorage with matching instance_id ('test-uuid-1'), mock fetch to return same id, init composable, verify state restores correctly

    **7. "computed properties"**
    - `isActive`: false for idle, true for welcome, true for workspace, true for backends.claude, false for complete
    - `currentStep`: 'idle' for idle, 'welcome' for welcome, 'workspace' for workspace, 'backends.claude' for backends.claude compound state
    - `canGoBack`: false for idle, false for welcome, true for workspace, true for backends.codex, false for complete
    - `canGoForward`: false for idle, true for welcome, true for workspace, true for verification, false for complete
    - `context`: returns default context initially, includes completedSteps after NEXT transitions

    **8. "event senders"**
    - `startTour()`: transitions from idle to welcome
    - `nextStep()`: transitions from welcome to workspace
    - `prevStep()`: transitions from workspace to welcome
    - `skipStep()`: transitions from welcome to workspace without marking complete
    - `completeTour()`: does nothing when canSkipAll is false (default guard)
    - `restartTour()`: returns to idle, clears localStorage
    - `clearTourState()`: removes localStorage item without sending events

    **9. "checkAndAutoAdvance"**
    - When on workspace and workspace is configured (mock /api/settings to return workspace_dir), auto-advances to backends
    - When on backends.claude and claude account exists (mock /admin/backends and /admin/backends/backend-claude), auto-advances to codex
    - When checks return false, stays on current step

    **10. "error resilience"**
    - Corrupt JSON in localStorage: set localStorage to invalid JSON string, init composable, verify starts at idle (no crash)
    - localStorage.setItem throws (quota exceeded simulation): mock localStorage.setItem to throw, verify transitions still work (persistence silently fails)

    IMPORTANT pitfalls to avoid (from 10-RESEARCH.md):
    - Always use `vi.resetModules()` in beforeEach — the singleton leaks otherwise
    - Always use dynamic `import()` after resetModules
    - Wait for async init before asserting on state: `await vi.waitFor()`
    - Mock `onUnmounted` via `vi.mock('vue', ...)` to avoid "no active component" errors
