  <action>
    Create `frontend/src/machines/__tests__/tourMachine.test.ts` following the XState v5 direct testing pattern (Pattern 1 from 10-RESEARCH.md). Use `createActor(tourMachine)` to create actors, send events, and assert on `actor.getSnapshot()`.

    Structure the test file with these describe blocks:

    **1. "initial state"** — verify actor starts in `idle`, context has default values (instanceId: null, schemaVersion: 1, completedSteps: [])

    **2. "forward navigation (NEXT)"** — test the complete forward path:
    - idle -> START -> welcome
    - welcome -> NEXT -> workspace (+ markStepCompleted adds 'welcome' to completedSteps)
    - workspace -> NEXT -> backends.claude (+ markStepCompleted adds 'workspace')
    - backends.claude -> NEXT -> backends.codex (+ markStepCompleted adds '{"backends":"claude"}')
    - backends.codex -> NEXT -> backends.gemini
    - backends.gemini -> NEXT -> backends.opencode
    - backends.opencode: NEXT on opencode has NO local NEXT handler; parent `backends` state NEXT fires -> verification (+ markStepCompleted adds '{"backends":"opencode"}' from parent handler)
    - verification -> NEXT -> complete (final state, status='done')

    **3. "backward navigation (BACK)"** — test every BACK transition:
    - workspace -> BACK -> welcome
    - backends.claude (reached via START, NEXT, NEXT) -> BACK on parent backends -> workspace
    - backends.codex -> BACK -> backends.claude
    - backends.gemini -> BACK -> backends.codex
    - backends.opencode -> BACK -> backends.gemini
    - verification -> BACK -> backends (re-enters at initial child 'claude')
    - idle: BACK event is ignored (no BACK handler)
    - welcome: BACK event is ignored (no BACK handler)
    - complete: BACK event is ignored (final state)

    **4. "skip navigation (SKIP)"** — test every SKIP transition:
    - welcome -> SKIP -> workspace (NO markStepCompleted)
    - workspace -> SKIP -> backends (NO markStepCompleted)
    - backends.claude -> SKIP -> backends.codex (NO markStepCompleted)
    - backends.codex -> SKIP -> backends.gemini
    - backends.gemini -> SKIP -> backends.opencode
    - backends parent -> SKIP -> verification (NO markStepCompleted)
    - verification -> SKIP -> complete
    - Verify completedSteps is empty after skip-all path

    **5. "SKIP_ALL global event"** — test the root-level SKIP_ALL:
    - Default: SKIP_ALL from any state does NOT transition (canSkipAll guard returns false)
    - With guard override: create actor from `tourMachine.provide({ guards: { canSkipAll: () => true } })`, start in welcome, send SKIP_ALL -> complete
    - Test from multiple states (welcome, workspace, backends.claude, verification)

    **6. "RESTART global event"** — test root-level RESTART:
    - From welcome: RESTART -> idle, context reset to initial values
    - From backends.codex: RESTART -> idle, completedSteps cleared
    - From verification: RESTART -> idle
    - Verify context is fully reset: instanceId=null, schemaVersion=1, completedSteps=[]

    **7. "markStepCompleted action"** — test the assign action:
    - After NEXT from welcome: completedSteps includes 'welcome'
    - After NEXT through multiple steps: completedSteps accumulates correctly
    - Duplicate prevention: advance to welcome, RESTART, START, advance from welcome again — should not duplicate 'welcome'

    **8. "clearProgress action"** — test via RESTART:
    - Advance several steps, RESTART: completedSteps=[], instanceId=null

    **9. "event rejection in invalid states"** — test that invalid events are silently ignored:
    - idle: NEXT, BACK, SKIP are ignored
    - complete: all events except RESTART are ignored (final state — actually nothing works on final states)
    - welcome: BACK is ignored

    **10. "hierarchical backends substates"** — test compound state specifics:
    - Entering backends starts at 'claude' (initial child)
    - BACK on verification goes to backends but re-enters at 'claude' (initial child), not at the substep the user left

    For each test: create a new actor, start it, send events, assert with `actor.getSnapshot().value` for state, `.context` for context, `.status` for done status. Stop actor after each test.

    IMPORTANT: Do NOT mock any XState internals. Test the machine as a black box. Do NOT import `@xstate/vue`. Use only `createActor` from `xstate`.
