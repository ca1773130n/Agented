<task type="auto">
  <name>Task 2: Install XState v5 and define tour state machine</name>
  <files>frontend/package.json, frontend/src/machines/tourMachine.ts</files>
  <action>
    1. Install dependencies:
       ```bash

       ```
       Do NOT remove driver.js from package.json yet — that cleanup happens in a later phase when the old tour code is fully replaced.

    2. Create `frontend/src/machines/tourMachine.ts` with XState v5 setup() API:

       Define the machine using `setup({...}).createMachine({...})` pattern (NOT v4's `createMachine` directly).

       **States (flat + hierarchical):**
       - `idle` — initial state, tour not started
       - `welcome` — welcome page shown
       - `workspace` — workspace setup step
       - `backends` — compound state with child substeps:
         - `claude` — Claude Code account registration (initial child)
         - `codex` — Codex/OpenAI account
         - `gemini` — Gemini account
         - `opencode` — OpenCode account
       - `verification` — harness verification step
       - `complete` — tour finished

       **Events:**
       - `START` — idle -> welcome
       - `NEXT` — advances to next step/substep
       - `BACK` — goes to previous step/substep
       - `SKIP` — skips current step (advances forward)
       - `SKIP_ALL` — jumps directly to complete (with a guard placeholder `canSkipAll`)
       - `RESTART` — any state -> idle (resets tour)

       **Context:**
       ```typescript
       {
         instanceId: string | null,      // backend instance ID for staleness detection
         schemaVersion: number,           // for persistence format versioning (start at 1)
         completedSteps: string[],        // track which steps were completed vs skipped
       }
       ```

       **Guards (declare in setup, implement as stubs returning false):**
       - `isWorkspaceConfigured` — will check if workspace is already set
       - `hasClaudeAccount` — will check if Claude backend account exists
       - `hasAnyBackend` — will check if at least one backend is registered
       - `canSkipAll` — will check if all required setup is already done

       **Actions (declare in setup, implement as stubs):**
       - `markStepCompleted` — adds current state to completedSteps
       - `clearProgress` — resets context to initial values
       - `persistState` — placeholder for localStorage persistence

       Export the machine definition as `tourMachine` (named export).
       Export the context type as `TourContext`.
       Export the event type union as `TourEvent`.

       CRITICAL: Use XState v5 patterns only:
       - `setup({ types: {}, guards: {}, actions: {} }).createMachine({...})` — NOT `createMachine({...})` alone
       - Guards referenced by string name in transitions — NOT inline functions
       - Context typing via `setup({ types: { context: {} as TourContext } })`
       - Do NOT use `interpret()` (v4) — that's `createActor()` in v5 (but don't create actor here, that's Plan 02)

    AVOID: Do NOT import from 'xstate/lib/*' (v4 deep imports). Do NOT use `Machine()` (v4 alias). Do NOT add `invoke` services yet — guards will be made async via `fromPromise` in Plan 02. Keep this file as pure machine definition with no side effects.
  </action>
