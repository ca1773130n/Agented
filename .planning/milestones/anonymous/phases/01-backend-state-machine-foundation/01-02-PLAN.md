  <name>Task 1: useTourMachine composable with persistence and instance_id validation</name>
  <files>frontend/src/composables/useTourMachine.ts</files>
  <action>
    Create `frontend/src/composables/useTourMachine.ts` — the Vue 3 composable that wraps the XState machine with persistence, staleness detection, and guard implementations.

    **Persistence layer:**
    - Storage key: `agented-tour-state`
    - On composable init:
      1. Read persisted data from localStorage (JSON parse, catch errors → null)
      2. Validate `schemaVersion` matches current (start at 1) — discard on mismatch
      3. Fetch `/health/instance-id` (no auth header needed)
      4. Compare fetched `instance_id` with stored `instanceId`:
         - Match → restore snapshot via `createActor(tourMachine, { snapshot: persisted.snapshot })`
         - Mismatch or no stored data → `createActor(tourMachine)` fresh, store new instanceId
      5. If fetch fails (network error), fall through to fresh start (safe default)
    - Subscribe to actor transitions via `actor.subscribe(snapshot => {...})`:
      - On every transition, persist to localStorage:
        ```json
        {
          "schemaVersion": 1,
          "instanceId": "<from context or fetched>",
          "snapshot": actor.getPersistedSnapshot()
        }
        ```
    - Expose `clearTourState()` function that removes localStorage key and sends RESTART event

    **Guard implementations (provide to machine via setup or actor options):**
    Guards are synchronous in XState v5. For async checks, use the pattern: guards return `false` by default, and the composable runs async checks on init then sends events to advance if checks pass.

    - `isWorkspaceConfigured`: Check by fetching `/admin/setup/status` (or similar workspace endpoint). Since guards must be sync, implement as: on machine init, fire an async check; if workspace is configured, send NEXT to skip past workspace step. Store result in a ref.
    - `hasClaudeAccount`: Check by fetching `/admin/backends` and filtering for type 'claude'. Same async-then-advance pattern.
    - `hasAnyBackend`: Check if `/admin/backends` returns non-empty list.
    - `canSkipAll`: True only if workspace configured AND at least one backend exists.

    The async guard pattern (from research): Rather than trying to make guards async, run the checks when entering a state (via `invoke` with `fromPromise`). Define invoked services on the machine states that need guard checks:
    - `workspace` state: invoke a promise that checks workspace status. On resolve with `{configured: true}`, auto-transition via NEXT.
    - `backends.claude` (and siblings): invoke a promise checking if that specific backend exists. On resolve with `{exists: true}`, auto-transition via NEXT.

    Update the machine definition in `tourMachine.ts` to add these `invoke` blocks if needed, OR implement the guard-check-then-advance pattern entirely in the composable. Prefer the composable approach to keep the machine definition pure.

    **Composable return value:**
    ```typescript
    {
      state: ComputedRef<...>,           // current machine state value
      context: ComputedRef<TourContext>, // current context
      send: (event: TourEvent) => void, // send events to actor
      isActive: ComputedRef<boolean>,    // tour is not idle and not complete
      currentStep: ComputedRef<string>,  // current state name (e.g., 'workspace', 'backends.claude')
      canGoBack: ComputedRef<boolean>,   // not at first step
      canGoForward: ComputedRef<boolean>,// not at last step
      startTour: () => void,             // sends START event
      nextStep: () => void,              // sends NEXT event
      prevStep: () => void,              // sends BACK event
      skipStep: () => void,              // sends SKIP event
      completeTour: () => void,          // sends SKIP_ALL or navigates to complete
      restartTour: () => void,           // clears state and sends RESTART
      clearTourState: () => void,        // removes localStorage only
    }
    ```

    Use `useActor` from `@xstate/vue` if it provides the right reactivity. If `useActor` from `@xstate/vue@3` doesn't accept a snapshot option, create the actor manually with `createActor()` and use Vue's `shallowRef` to track state reactively via `actor.subscribe()`.

    AVOID:
    - Do NOT import from the old `useTour.ts` — this is a parallel implementation that will replace it in Phase 3.
    - Do NOT use `interpret()` (v4 API).
    - Do NOT make guards async — use the invoke-or-check-then-advance pattern.
    - Do NOT store the full XState snapshot as a giant JSON blob — use `getPersistedSnapshot()` which produces a minimal serializable form.
    - API calls for guard checks should include the API key from localStorage (`X-API-Key` header) if available, but gracefully handle 401/403 by treating the check as "not configured" (falling through to show the step).
