  <files>frontend/src/App.vue, frontend/src/components/tour/TourSpotlight.vue, frontend/src/components/tour/TourTooltip.vue</files>
  <action>
    Three changes are needed to fully honor prefers-reduced-motion: reduce:

    **1. App.vue — add --tour-transition-speed override inside the existing reduced-motion block**

    The existing block at the bottom of App.vue sets transition-duration to 0.01ms globally, but TourTooltip reads `--tour-transition-speed` as a CSS custom property. Add it explicitly inside the existing `@media (prefers-reduced-motion: reduce)` block:

    ```css
    @media (prefers-reduced-motion: reduce) {
      /* existing * rules ... */
      :root {
        --tour-transition-speed: 0ms;
      }
    }
    ```

    **2. TourSpotlight.vue — add animation: none for .tour-spotlight-glow in reduced-motion media query**

    The global `animation-duration: 0.01ms` technically stops animation but leaves it running at near-zero duration with `animation-iteration-count: 1`. Add an explicit `animation: none` in a scoped media query for the glow element to cleanly disable it. Add at the bottom of the `<style scoped>` block:

    ```css
    @media (prefers-reduced-motion: reduce) {
      .tour-spotlight-glow {
        animation: none;
      }
      .tour-spotlight {
        transition: none;
      }
    }
    ```

    **3. TourTooltip.vue — bypass the 200ms setTimeout when reduced motion is active**

    The `watch(() => props.targetRect, ...)` uses `setTimeout(..., 200)` for the two-phase fade-out/in transition. When reduced motion is active, this 200ms wait is unnecessary and causes a brief blank period. Replace the hardcoded `200` with a dynamic duration check:

    In the script setup, add a helper after the existing imports:
    ```ts
    function getTransitionDuration(): number {
      const raw = getComputedStyle(document.documentElement)
        .getPropertyValue('--tour-transition-speed')
        .trim()
      // e.g. "200ms" → 200, "0ms" → 0
      return parseInt(raw, 10) || 0
    }
    ```

    Then in the `watch(() => props.targetRect, ...)` handler, replace `setTimeout(() => { ... }, 200)` with `setTimeout(() => { ... }, getTransitionDuration())`.

    Also add reduced-motion override at the bottom of TourTooltip's `<style scoped>`:
    ```css
    @media (prefers-reduced-motion: reduce) {
      .tour-tooltip {
        transition: none;
      }
    }
    ```

  <action>
    **TourOverlay.vue — add ARIA live region**

    Add a visually-hidden `aria-live="polite"` element to the TourOverlay template. It must be visually hidden (not `display:none` or `v-show=false`, or screen readers will not read it). Use the `.sr-only` visually-hidden pattern:

    In `<script setup>`, add a computed for the announcement text that updates whenever step or stepNumber/totalSteps changes:

    ```ts
    const announcement = computed(() => {
      if (!props.active || !props.step) return ''
      return `Step ${props.stepNumber} of ${props.totalSteps}: ${props.step.title}. ${props.step.message}`
    })
    ```

    In `<template>`, add immediately inside the root `<div>` (or as a sibling — must be present in DOM when active=true):

    ```html
    <!-- OB-38: ARIA live announcements for screen readers -->
    <div
      aria-live="polite"
      aria-atomic="true"
      class="sr-only"
    >{{ announcement }}</div>
    ```

    Add the `.sr-only` class in `<style scoped>` (standard visually-hidden pattern):

    ```css
    .sr-only {
      position: absolute;
      width: 1px;
      height: 1px;
      padding: 0;
      margin: -1px;
      overflow: hidden;
      clip: rect(0, 0, 0, 0);
      white-space: nowrap;
      border: 0;
    }
    ```
