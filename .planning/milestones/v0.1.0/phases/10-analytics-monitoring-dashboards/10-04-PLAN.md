    **API client:**

    Create `frontend/src/services/api/analytics.ts` following the established pattern from budgets.ts:

    ```typescript
    import { apiFetch } from './client';

    export const analyticsApi = {
      fetchCostAnalytics: (params?: {
        group_by?: string;
        start_date?: string;
        end_date?: string;
        entity_type?: string;
      }) => {
        const query = new URLSearchParams();
        if (params?.group_by) query.set('group_by', params.group_by);
        if (params?.start_date) query.set('start_date', params.start_date);
        if (params?.end_date) query.set('end_date', params.end_date);
        if (params?.entity_type) query.set('entity_type', params.entity_type);
        return apiFetch<CostAnalyticsResponse>(`/admin/analytics/cost?${query}`);
      },

      fetchExecutionAnalytics: (params?: {
        group_by?: string;
        start_date?: string;
        end_date?: string;
        trigger_id?: string;
      }) => { /* same pattern */ },

      fetchEffectiveness: (params?: {
        trigger_id?: string;
        group_by?: string;
        start_date?: string;
        end_date?: string;
      }) => apiFetch<EffectivenessResponse>(`/admin/analytics/effectiveness?${query}`),
    };
    ```

    **Types:**

    Add to `frontend/src/services/api/types.ts`:
    ```typescript
    // Analytics types (ANA-01, ANA-02, ANA-03)
    export interface CostDataPoint {
      entity_id: string;
      period: string;
      total_cost_usd: number;
      total_tokens: number;
    }

    export interface CostAnalyticsResponse {
      data: CostDataPoint[];
      period_count: number;
      total_cost: number;
    }

    export interface ExecutionDataPoint {
      period: string;
      total_executions: number;
      success_count: number;
      failed_count: number;
      cancelled_count: number;
      avg_duration_ms: number | null;
      backend_type: string;
    }

    export interface ExecutionAnalyticsResponse {
      data: ExecutionDataPoint[];
      period_count: number;
      total_executions: number;
    }

    export interface EffectivenessResponse {
      total_reviews: number;
      accepted: number;
      ignored: number;
      pending: number;
      acceptance_rate: number;
      over_time: Array<{
        period: string;
        total_reviews: number;
        accepted: number;
        acceptance_rate: number;
      }>;
    }
    ```

    **Re-export:**

    Add `export { analyticsApi } from './analytics';` to `frontend/src/services/api/index.ts`.
    Add re-exports for the analytics types from `types.ts` in `index.ts`.

    AVOID: Using `any` types — all API responses must be typed. AVOID: Building a custom HTTP client — use existing `apiFetch`.

    Create the `frontend/src/components/analytics/` directory with 4 chart components. Each follows the established pattern from TokenUsageChart.vue and PrHistoryChart.vue:

    **Standard chart component pattern (apply to all 4):**
    - Import `Chart, registerables` from 'chart.js' and register them. Import 'chartjs-adapter-date-fns'.
    - Accept `data` prop with the appropriate typed array.
    - Use `ref<HTMLCanvasElement>` for canvas element.
    - In `onMounted()`, create Chart instance. In `onUnmounted()`, call `chart.destroy()` to prevent memory leaks (10-RESEARCH.md Pitfall 2).
    - Watch `data` prop — on change, destroy old chart and create new one.
    - Use dark theme colors: `{ grid: 'rgba(255, 255, 255, 0.06)', text: 'rgba(255, 255, 255, 0.5)' }` for scales. Use `responsive: true, maintainAspectRatio: false`.
    - Canvas wrapped in a div with `height: 300px`.

    **1. CostTrendChart.vue** — Line chart showing cost over time.
    - Props: `data: CostDataPoint[]`
    - X-axis: period (time scale). Y-axis: total_cost_usd.
    - Color: `#8855ff` (--accent-violet) for the line with `rgba(136, 85, 255, 0.1)` fill.
    - If multiple entities in data, group by entity_id and show multiple lines with different colors.

    **2. ExecutionVolumeChart.vue** — Stacked bar chart showing execution counts.
    - Props: `data: ExecutionDataPoint[]`
    - X-axis: period. Y-axis: count.
    - 3 stacked datasets: success (#00ff88, --accent-emerald), failed (#ff4444, --accent-crimson), cancelled (#ffaa00, --accent-amber).
    - Tooltip shows total + breakdown.

    **3. SuccessRateChart.vue** — Line chart showing success rate percentage over time.
    - Props: `data: ExecutionDataPoint[]`
    - Compute success_rate = success_count / total_executions * 100 for each period.
    - X-axis: period. Y-axis: percentage (0-100).
    - Color: `#00d4ff` (--accent-cyan) for the line.
    - Add a horizontal reference line at 80% (dashed, subtle) as a "healthy" baseline.

    **4. BotEffectivenessChart.vue** — Doughnut chart for overall + line chart for over_time.
    - Props: `summary: { accepted, ignored, pending, acceptance_rate }`, `overTime: Array<{ period, acceptance_rate }>`
    - Doughnut: 3 segments (accepted=#00ff88, ignored=#ff4444, pending=#ffaa00).
    - Below doughnut: large acceptance_rate percentage text.
    - Line chart: acceptance_rate over time with #8855ff line.
