<objective>
Implement an encrypted secrets vault using Fernet symmetric encryption for storing API keys, tokens, and credentials at rest, with audit logging on every access.

Purpose: Secrets vault is required for integration adapters (Slack tokens, JIRA API keys) in Plan 04, and for GitOps configuration in Plan 05. Every secret access must be audit-logged for compliance.

Output: `secret_vault_service.py` with Fernet encryption, `secrets` DB table, management API routes, comprehensive tests.

Research basis: Fernet authenticated encryption (cryptography.io docs), MultiFernet for key rotation (NIST SP 800-132), audit-on-access pattern from 11-RESEARCH.md.
</objective>

       ```sql
       CREATE TABLE IF NOT EXISTS secrets (
           id TEXT PRIMARY KEY,
           name TEXT NOT NULL UNIQUE,
           encrypted_value TEXT NOT NULL,
           description TEXT DEFAULT '',
           scope TEXT DEFAULT 'global',
           created_by TEXT DEFAULT 'system',
           created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
           updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
           last_accessed_at TIMESTAMP
       );
       CREATE INDEX IF NOT EXISTS idx_secrets_name ON secrets(name);
       CREATE INDEX IF NOT EXISTS idx_secrets_scope ON secrets(scope);
       ```

    2. **ID generation** -- Add to `ids.py`:

       - `generate_secret_id()` and `_get_unique_secret_id(conn)` functions.

    3. **DB CRUD** -- Create `backend/app/db/secrets.py`:
       - `create_secret(name, encrypted_value, description, scope, created_by) -> str` (returns secret ID)
       - `get_secret(secret_id) -> Optional[dict]`
       - `get_secret_by_name(name) -> Optional[dict]`
       - `list_secrets() -> list[dict]` -- returns metadata only (id, name, description, scope, created_at), NEVER encrypted_value
       - `update_secret(secret_id, encrypted_value=None, description=None) -> bool`
       - `delete_secret(secret_id) -> bool`
       - `update_last_accessed(secret_id) -> bool`
       - Re-export from `db/__init__.py`.

    4. **Pydantic models** -- Create `backend/app/models/secret.py`:
       - `SecretCreate(name: str, value: str, description: str = "", scope: str = "global")` -- value is plaintext, encrypted by service
       - `SecretUpdate(value: Optional[str], description: Optional[str])`
       - `SecretResponse(id, name, description, scope, created_by, created_at, updated_at, last_accessed_at)` -- NO value field
       - `SecretPath(secret_id: str)` for path params

    5. **Secret vault service** -- Create `backend/app/services/secret_vault_service.py`:
       - Class-level `_fernet: Optional[MultiFernet] = None` with lazy initialization
       - `_get_fernet() -> MultiFernet`:
         a. Read `AGENTED_VAULT_KEYS` env var (comma-separated Fernet keys)
         b. If not set, log warning and raise `RuntimeError("AGENTED_VAULT_KEYS not configured")`
         c. Create `MultiFernet([Fernet(k.strip()) for k in keys_str.split(",")])`
         d. Cache in `_fernet` class attribute
       - `encrypt(plaintext: str) -> str` -- returns base64-encoded ciphertext
       - `decrypt(ciphertext: str, purpose: str = "", accessor: str = "") -> str`:
         a. Call `AuditLogService.log(action="secret.access", entity_type="secret", entity_id=purpose, outcome="accessed", details={"accessor": accessor})`
         b. Decrypt and return plaintext
       - `create_secret(name, value, description, scope, created_by) -> str`:
         a. Encrypt value
         b. Store via `db.secrets.create_secret()`
         c. Audit log the creation (NOT the value)
       - `get_secret_value(secret_id_or_name, accessor) -> str`:
         a. Look up secret by ID or name
         b. Decrypt encrypted_value
         c. Update last_accessed_at
         d. Audit log the access
         e. Return plaintext
       - `get_secrets_for_execution(scope="global") -> dict[str, str]`:
         a. List secrets matching scope
         b. Decrypt each, returning `{"AGENTED_SECRET_{name}": plaintext}` dict
         c. This is used to inject env vars at execution time
       - `is_configured() -> bool` -- checks if AGENTED_VAULT_KEYS is set
       - `rotate_key(new_key: str)` -- re-encrypt all secrets with new primary key (for future use)

    Avoid: NEVER store the encryption key in the database (see 11-RESEARCH.md anti-pattern). NEVER include plaintext or encrypted values in API response JSON. NEVER store a plaintext fallback "for debugging" (11-RESEARCH.md Pitfall). Use `AGENTED_SECRET_` prefix naming convention for injected env vars and strip them from log output.
