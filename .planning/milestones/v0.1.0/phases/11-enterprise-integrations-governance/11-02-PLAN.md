<objective>
Implement an encrypted secrets vault using Fernet symmetric encryption for storing API keys, tokens, and credentials at rest, with audit logging on every access, and wire secret injection into the execution service subprocess environment.

Purpose: Secrets vault is required for integration adapters (Slack tokens, JIRA API keys) in Plan 04, and for GitOps configuration in Plan 05. Every secret access must be audit-logged for compliance. Secrets must be injected into execution subprocess env for bots to use.

Output: `secret_vault_service.py` with Fernet encryption, `secrets` DB table, management API routes, ExecutionService integration for env injection, comprehensive tests.

Research basis: Fernet authenticated encryption (cryptography.io docs), MultiFernet for key rotation (NIST SP 800-132), audit-on-access pattern from 11-RESEARCH.md.
</objective>

    5. **Secret vault service** -- Create `backend/app/services/secret_vault_service.py`:
       - Class-level `_fernet: Optional[MultiFernet] = None` with lazy initialization
       - `_get_fernet() -> MultiFernet`:
         a. Read `AGENTED_VAULT_KEYS` env var (comma-separated Fernet keys)
         b. If not set, log warning and raise `RuntimeError("AGENTED_VAULT_KEYS not configured")`
         c. Create `MultiFernet([Fernet(k.strip()) for k in keys_str.split(",")])`
         d. Cache in `_fernet` class attribute
       - `encrypt(plaintext: str) -> str` -- returns base64-encoded ciphertext
       - `decrypt(ciphertext: str, purpose: str = "", accessor: str = "") -> str`:
         a. Call `AuditLogService.log(action="secret.access", entity_type="secret", entity_id=purpose, outcome="accessed", details={"accessor": accessor})`
         b. Decrypt and return plaintext
       - `create_secret(name, value, description, scope, created_by) -> str`:
         a. Encrypt value
         b. Store via `db.secrets.create_secret()`
         c. Audit log the creation (NOT the value)
       - `update_secret(secret_id, value=None, description=None, actor="system") -> bool`:
         a. Load existing secret metadata (for before/after comparison)
         b. If value provided, encrypt new value
         c. Update via `db.secrets.update_secret()`
         d. Call `AuditLogService.log_field_changes(action="secret.update", entity_type="secret", entity_id=secret_id, old_entity=old_metadata, new_entity=new_metadata)` for description/scope changes
         e. If value changed, audit log `action="secret.value_rotated"` (without logging the actual value)
       - `delete_secret(secret_id, actor="system") -> bool`:
         a. Load existing secret metadata (for before state)
         b. Delete via `db.secrets.delete_secret()`
         c. Call `AuditLogService.log_field_changes(action="secret.delete", entity_type="secret", entity_id=secret_id, old_entity=metadata, new_entity={})` to record the deletion with before state
       - `get_secret_value(secret_id_or_name, accessor) -> str`:
         a. Look up secret by ID or name
         b. Decrypt encrypted_value
         c. Update last_accessed_at
         d. Audit log the access
         e. Return plaintext
       - `get_secrets_for_execution(scope="global") -> dict[str, str]`:
         a. List secrets matching scope
         b. Decrypt each, returning `{"AGENTED_SECRET_{name}": plaintext}` dict
         c. This is used to inject env vars at execution time
       - `is_configured() -> bool` -- checks if AGENTED_VAULT_KEYS is set
       - `rotate_key(new_key: str)` -- re-encrypt all secrets with new primary key (for future use)

    Avoid: NEVER store the encryption key in the database (see 11-RESEARCH.md anti-pattern). NEVER include plaintext or encrypted values in API response JSON. NEVER store a plaintext fallback "for debugging" (11-RESEARCH.md Pitfall). Use `AGENTED_SECRET_` prefix naming convention for injected env vars and strip them from log output.

       - In the `_execute_trigger()` method (or equivalent method that builds `proc_env`), BEFORE the `subprocess.Popen` call:
         ```python
         # Inject secrets from vault into subprocess environment
         try:
             from app.services.secret_vault_service import SecretVaultService
             if SecretVaultService.is_configured():
                 vault_secrets = SecretVaultService.get_secrets_for_execution(scope="global")
                 if vault_secrets:
                     if env_overrides is None:
                         env_overrides = {}
                     env_overrides.update(vault_secrets)
         except Exception as e:
             logger.warning("Failed to inject vault secrets into execution env: %s", e)
         ```
       - This merges `AGENTED_SECRET_*` env vars into the existing `env_overrides` dict that gets passed to `proc_env`
       - The try/except ensures vault failures NEVER break execution (graceful degradation)
