    5. **Adapter base** -- Create `backend/app/services/integrations/__init__.py`:
       ```python
       from abc import ABC, abstractmethod
       from typing import Dict, Optional, Tuple

       class IntegrationAdapter(ABC):
           @abstractmethod
           def send_notification(self, channel: str, message: str, metadata: dict = None) -> bool: ...

           @abstractmethod
           def create_ticket(self, project_key: str, finding: dict) -> Optional[str]: ...

           @abstractmethod
           def validate_config(self, config: dict) -> Tuple[bool, Optional[str]]: ...

       ADAPTER_REGISTRY: Dict[str, type] = {}

       def register_adapter(adapter_type: str, adapter_class: type):
           ADAPTER_REGISTRY[adapter_type] = adapter_class

       def get_adapter(adapter_type: str, **kwargs) -> Optional[IntegrationAdapter]:
           cls = ADAPTER_REGISTRY.get(adapter_type)
           if cls:
               return cls(**kwargs)
           return None
       ```

    6. **Slack adapter** -- `backend/app/services/integrations/slack_adapter.py`:
       - Constructor takes `token: str` (retrieved from secrets vault by caller)
       - `send_notification()`: Use `slack_sdk.WebClient.chat_postMessage()` with Block Kit formatting (see research example). Format execution summary with status emoji, bot name, duration.
       - `create_ticket()`: Return None (Slack doesn't create tickets)
       - `validate_config()`: Check token is non-empty, optionally call `auth.test` API to verify scopes
       - Register via `register_adapter("slack", SlackAdapter)` at module level
       - Handle `SlackApiError` gracefully with logging

    7. **Teams adapter** -- `backend/app/services/integrations/teams_adapter.py`:
       - Constructor takes `webhook_url: str`
       - `send_notification()`: POST MessageCard JSON to webhook_url via `httpx.post()` with 10s timeout (see research example)
       - `create_ticket()`: Return None (Teams doesn't create tickets)
       - `validate_config()`: Check webhook_url is non-empty and starts with https://
       - Register via `register_adapter("teams", TeamsAdapter)`

    8. **JIRA adapter** -- `backend/app/services/integrations/jira_adapter.py`:
       - Constructor takes `server: str, email: str, api_token: str`
       - `send_notification()`: Return False (JIRA doesn't support notifications)
       - `create_ticket()`: Use `jira.JIRA.create_issue()` with severity-to-priority mapping (see research example). Map: critical->Highest, high->High, medium->Medium, low->Low. Add labels ["agented", "automated"].
       - `validate_config()`: Check server, email, api_token all non-empty
       - Register via `register_adapter("jira", JiraAdapter)`

    9. **Linear adapter** -- `backend/app/services/integrations/linear_adapter.py`:
       - Constructor takes `api_key: str`
       - Use raw `httpx` with GraphQL mutations (no Python SDK per research recommendation)
       - `send_notification()`: Return False
       - `create_ticket()`: POST GraphQL mutation `issueCreate` to `https://api.linear.app/graphql` with Authorization bearer header. Include title, description, priority mapping.
       - `validate_config()`: Check api_key non-empty
       - Register via `register_adapter("linear", LinearAdapter)`

    Avoid: Do NOT store credentials in the integrations DB table -- store only a `secret_name` reference; actual credentials come from SecretVaultService. Do NOT make synchronous external API calls in request handlers -- all adapter calls should happen in background threads (consistent with existing _stream_pipe pattern). Per 11-RESEARCH.md, use `slack-sdk` not raw HTTP for Slack (it handles rate limiting, retry, token rotation).
