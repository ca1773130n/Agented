<objective>
Implement bot configuration export/import as YAML/JSON (with upsert support for GitOps) and execution bookmarking with deep-links, notes, and tags.

Purpose: Config export/import is the foundation for GitOps sync (Plan 05) and enables configuration-as-code. The upsert flag on import_trigger() is critical for GitOps to update existing triggers by name without creating duplicates. Bookmarking provides execution pinning for bot profile pages, a self-contained user-facing feature.

Output: `config_export_service.py` for YAML/JSON config management with upsert, `bookmark_service.py` for execution pinning, API routes for both, tests including roundtrip verification.

Research basis: Existing `SuperAgentExportService` pattern for export/import, `pyyaml` (already a dependency), deep-link format from 11-RESEARCH.md.
</objective>

    Based on existing `SuperAgentExportService` pattern and 11-RESEARCH.md config export example:

    1. **Pydantic models** -- Create `backend/app/models/config_export.py`:
       - `ConfigExportResponse(version: str, kind: str, metadata: dict, spec: dict)`
       - `ConfigImportRequest(config: str, format: str = "yaml", upsert: bool = False)` -- accepts YAML or JSON string, optional upsert flag
       - `ConfigImportResult(trigger_id: str, name: str, status: str, message: str)` -- status is "created" or "updated"

    2. **Config export service** -- Create `backend/app/services/config_export_service.py`:
       - `export_trigger(trigger_id, format="yaml") -> str`:
         a. Load trigger from DB via `get_trigger(trigger_id)`
         b. Load associated paths via `list_paths_for_trigger(trigger_id)`
         c. Build config dict with structure:
            ```python
            {
                "version": "1.0",
                "kind": "trigger",
                "metadata": {
                    "name": trigger["name"],
                    "backend_type": trigger["backend_type"],
                    "trigger_source": trigger["trigger_source"],
                    "exported_at": iso_timestamp,
                },
                "spec": {
                    "prompt_template": trigger["prompt_template"],
                    "model": trigger.get("model"),
                    "execution_mode": trigger.get("execution_mode", "direct"),
                    "timeout_seconds": trigger.get("timeout_seconds"),
                    "allowed_tools": trigger.get("allowed_tools"),
                    "enabled": bool(trigger.get("enabled", 1)),
                    "detection_keyword": trigger.get("detection_keyword", ""),
                    "match_field_path": trigger.get("match_field_path"),
                    "match_field_value": trigger.get("match_field_value"),
                    "text_field_path": trigger.get("text_field_path"),
                    "paths": [{"local": p["local_project_path"], "type": p["path_type"],
                               "github_repo_url": p.get("github_repo_url")}
                              for p in paths],
                },
            }
            ```
         d. If scheduled trigger, include schedule config under `spec.schedule`
         e. Return `yaml.dump(config, default_flow_style=False, sort_keys=False)` or `json.dumps(config, indent=2)`

       - `import_trigger(config_str, format="yaml", upsert=False) -> tuple[str, str]`:
         a. Parse YAML or JSON string
         b. Validate version and kind
         c. If `upsert=True`:
            - Look up existing trigger by `metadata.name` match
            - If found: update existing trigger fields from parsed spec, update associated paths, audit log the update with before/after state
            - Return `(trigger_id, "updated")`
         d. If `upsert=False` or no existing trigger found:
            - Create new trigger via `create_trigger()` from parsed spec
            - Create associated paths
            - Return `(trigger_id, "created")`
         e. Audit log the import

       - `export_all_triggers(format="yaml") -> str`:
         a. Export all triggers as a multi-document YAML stream or JSON array

       - `validate_config(config_str, format="yaml") -> tuple[bool, Optional[str]]`:
         a. Parse and validate structure without importing
         b. Return (valid, error_message)

    3. **Routes** -- Create `backend/app/routes/config_export.py`:
       - `APIBlueprint("config_export", __name__, url_prefix="/admin", abp_tags=[Tag(name="config")])`
       - `GET /admin/triggers/<trigger_id>/export?format=yaml` -- export single trigger
       - `POST /admin/triggers/import` -- import trigger from YAML/JSON body (accepts `upsert` query param or body field)
       - `POST /admin/triggers/validate-config` -- validate config without importing
       - `GET /admin/triggers/export-all?format=yaml` -- export all triggers
       - Register in `routes/__init__.py` and add to rate limiting list

    4. **Tests** -- Create `backend/tests/test_config_export.py`:
       - Test export produces valid YAML (parse roundtrip)
       - Test export produces valid JSON
       - Test import creates identical trigger (field-by-field comparison, excluding id and timestamps)
       - **Test lossless roundtrip: seed trigger -> export -> import(upsert=False on clean DB) -> export -> compare normalized YAML (strip exported_at timestamps before comparison)**
       - Test validate_config catches malformed input
       - Test import with missing required fields returns error
       - Test scheduled trigger export includes schedule config
       - **Test upsert=True: create trigger, export, modify YAML, import with upsert=True, verify trigger updated (not duplicated)**
       - **Test upsert=False with existing name: creates new trigger (no update)**
       - **Test deep-link URL resolution: create bookmark with line_number, verify resolve_deep_link() returns correct URL format**
       - Use `isolated_db` fixture with seeded test trigger

    Avoid: Do NOT export sensitive fields (webhook_secret) in config -- filter them out. Do NOT use `yaml.load()` without `Loader=yaml.SafeLoader` (security risk). The `pyyaml` library is already a dependency, do not add a new one.

       ```sql
       CREATE TABLE IF NOT EXISTS bookmarks (
           id TEXT PRIMARY KEY,
           execution_id TEXT NOT NULL,
           trigger_id TEXT NOT NULL,
           title TEXT NOT NULL DEFAULT '',
           notes TEXT DEFAULT '',
           tags TEXT DEFAULT '',
           line_number INTEGER,
           deep_link TEXT NOT NULL,
           created_by TEXT DEFAULT 'system',
           created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
       );
       CREATE INDEX IF NOT EXISTS idx_bookmarks_trigger ON bookmarks(trigger_id);
       CREATE INDEX IF NOT EXISTS idx_bookmarks_execution ON bookmarks(execution_id);
       ```

    2. **ID generation** -- Add to `ids.py`:
