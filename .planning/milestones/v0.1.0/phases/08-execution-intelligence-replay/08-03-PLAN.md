<objective>
Implement live collaborative execution viewer with presence indicators and inline commenting (EXE-05) as backend service, database layer, and API endpoints.

Purpose: Enables multiple team members to watch the same live execution stream simultaneously with real-time presence awareness (who is watching) and inline commenting on specific log lines. Comments persist beyond the session so they are visible when reviewing execution history.

Output: CollaborativeViewerService, viewer_comments DB table, and API endpoints for presence management and inline commenting.

Research basis: Extends the existing SSE fan-out pattern (ExecutionLogService._subscribers with threading.Queue per subscriber) with additional event types (presence_join, presence_leave, presence_heartbeat, inline_comment) per 08-RESEARCH.md Recommendation 4. Uses companion POST endpoints for client-to-server communication (comments) since SSE is unidirectional, following the standard pattern documented in Flask-SSE and Ajackus SSE guides.
</objective>

         ```python
         _viewers: Dict[str, Dict[str, dict]] = {}  # {execution_id: {viewer_id: {"name": str, "joined_at": str, "last_seen": datetime}}}
         _lock = threading.Lock()
         HEARTBEAT_INTERVAL = 30  # seconds
         HEARTBEAT_MISS_THRESHOLD = 2  # misses before removal
         ```

       - `join(execution_id: str, viewer_id: str, viewer_name: str) -> dict` method:
         a. Register viewer in _viewers dict under the execution_id
         b. Broadcast `presence_join` event via ExecutionLogService._broadcast() with viewer list
         c. Return current viewer list for the execution

       - `leave(execution_id: str, viewer_id: str) -> None` method:
         a. Remove viewer from _viewers dict
         b. Broadcast `presence_leave` event with updated viewer list
         c. Clean up empty execution entries

         a. Update last_seen timestamp for the viewer

       - `get_viewers(execution_id: str) -> list[dict]` method:
         a. Return list of current viewers for the execution

       - `cleanup_stale_viewers() -> int` method:
         a. Iterate all execution viewer dicts
         b. Remove viewers whose last_seen is older than HEARTBEAT_INTERVAL * HEARTBEAT_MISS_THRESHOLD
         c. Broadcast `presence_leave` for each removed viewer
         d. Return count of removed viewers
         e. Per 08-RESEARCH.md Pitfall 3: "Remove viewers that miss 2+ heartbeats. The existing ExecutionLogService.cleanup_stale_executions() pattern should be extended."

       - `post_comment(execution_id: str, viewer_id: str, viewer_name: str, line_number: int, content: str) -> dict` method:
         a. Persist comment to DB via create_viewer_comment()
         b. Broadcast `inline_comment` event via ExecutionLogService._broadcast() with comment data
         c. Return created comment dict (id, viewer_id, viewer_name, line_number, content, created_at)
         d. Per 08-RESEARCH.md Open Question 3: "Anchor comments to stdout line numbers (immutable once execution completes)"

       - `get_execution_comments(execution_id: str) -> list[dict]` method:
         a. Return all persisted comments for the execution from DB
         b. This enables viewing comments on historical executions (not just live SSE)
