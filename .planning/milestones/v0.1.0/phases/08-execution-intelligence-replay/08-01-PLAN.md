<objective>
Implement execution replay with side-by-side diff comparison (EXE-01) and diff-aware context injection for PR-triggered bots (EXE-02) as backend services, database layer, and API endpoints.

Purpose: Enables A/B testing of prompt revisions and regression detection by replaying executions with identical inputs and viewing output differences. Diff-aware context reduces token costs by 40-80% for PR reviews by extracting only changed files instead of full file contents.

Output: ReplayService, DiffContextService, replay_comparisons DB table, replay API endpoints, and the unidiff dependency installed.

Research basis: Python stdlib difflib for output comparison (08-RESEARCH.md Recommendation 5), unidiff v0.7.5 for diff parsing (Recommendation 2). Replay uses stored prompt/command from execution_logs (Pattern 1: Input Snapshot + Re-execution). Diff-aware context validated by LongLLMLingua (Jiang 2023) showing 4x token reduction with quality improvement.
</objective>

    1. Create `backend/app/routes/replay.py` as an APIBlueprint following existing route patterns (see executions.py, workflows.py):
       - Tag: `replay` with description "Execution replay and A/B comparison"
       - URL prefix: `/admin`

       Endpoints:
       a. `POST /admin/executions/<execution_id>/replay` -- Replay an execution
          - Accepts optional JSON body with `notes` field
          - Calls `ReplayService.replay_execution(execution_id, notes)`
          - Returns 201 with {comparison_id, original_execution_id, replay_execution_id}
          - Returns 404 if original execution not found
          - Returns 400 if original execution is still running

       b. `GET /admin/executions/<execution_id>/comparisons` -- List replay comparisons for an execution
          - Calls `get_replay_comparisons_for_execution(execution_id)`
          - Returns 200 with list of comparisons

       c. `GET /admin/replay-comparisons/<comparison_id>` -- Get a specific comparison
          - Calls `get_replay_comparison(comparison_id)`
          - Returns 200 with comparison details
          - Returns 404 if not found

       d. `GET /admin/replay-comparisons/<comparison_id>/diff` -- Get side-by-side output diff
          - Fetches comparison, then calls `ReplayService.compare_outputs(original_id, replay_id)`
          - Returns 200 with OutputDiff structure (diff_lines, change_summary, line counts)
          - Returns 404 if comparison not found
          - Returns 409 if either execution is still running (diff requires completed output)

       e. `POST /admin/diff-context/preview` -- Preview diff-aware context extraction
          - Accepts JSON body with `diff_text: str` and optional `context_lines: int`
          - Calls `DiffContextService.extract_pr_diff_context(diff_text, context_lines)`
          - Returns 200 with {context: str, token_estimate: dict}
          - This endpoint enables testing diff-aware context without a real PR
