  <name>Task 1: Prompt version history component and webhook test console component integrated into trigger dashboard</name>
  <files>frontend/src/components/triggers/PromptVersionHistory.vue, frontend/src/components/triggers/WebhookTestConsole.vue, frontend/src/views/GenericTriggerDashboard.vue</files>
  <action>
    Following the existing trigger component patterns and dark theme:

    1. Create `frontend/src/components/triggers/PromptVersionHistory.vue`:
       - Props: `triggerId: string`
       - On mount: call `triggerApi.getPromptHistory(triggerId)` to load version history
       - Template structure:
         a. Section header: "Prompt Version History"
         b. Version list (reverse chronological):
            - Each entry shows: version number (derived from position), author, timestamp (formatted with date-fns), change summary
            - Expandable diff view: display the `diff_text` field in a `<pre>` block with monospace font
            - Diff lines should be color-coded: lines starting with `+` in green, `-` in red, `@@` in blue/cyan (use CSS classes)
            - "Rollback to this version" button on each entry (except the current/latest)
         c. Rollback confirmation: when clicked, confirm with dialog, then call `triggerApi.rollbackPrompt(triggerId, versionId)`
         d. After rollback: show success toast, reload history list
         e. Empty state: "No version history yet. Edit the prompt template to start tracking changes."
       - Loading/error states with standard patterns

    2. Create `frontend/src/components/triggers/WebhookTestConsole.vue`:
       - Props: `triggerId: string`
       - Template structure:
         a. Section header: "Webhook Payload Test Console"
         b. Left panel: JSON payload editor
            - `<textarea>` with monospace font, pre-filled with example JSON:
              ```json
              {
                "message": "Test webhook message",
                "paths": "/path/to/project",
                "event": {}
              }
              ```
            - JSON validation indicator (green check / red X based on whether content parses as valid JSON)
            - "Preview" button (disabled if JSON is invalid)
         c. Right panel: Preview results (shown after clicking Preview)
            - "Rendered Prompt" section: display rendered_prompt in a styled code block
            - "CLI Command" section: display cli_command in a monospace code block
            - "Backend" section: show backend_type and model
            - "Unresolved Placeholders" warning: if unresolved_placeholders is non-empty, list them with a yellow warning icon
            - "Unresolved Snippets" warning: if unresolved_snippets is non-empty, list them with a yellow warning icon
         d. On "Preview" click: call `triggerApi.previewPromptFull(triggerId, parsedPayload)`
         e. Loading state during preview fetch
         f. Prominent note: "This is a dry-run preview. No bot execution will be triggered."
       - CRITICAL: This component must ONLY call the preview-prompt-full API endpoint. It must NOT call any execution or run endpoint.

    3. Integrate both components into `frontend/src/views/GenericTriggerDashboard.vue`:
       - Add tab navigation or collapsible sections for "Version History" and "Test Console"
       - Import and render `<PromptVersionHistory :trigger-id="triggerId" />` and `<WebhookTestConsole :trigger-id="triggerId" />`
       - These should appear as additional sections/tabs on the trigger detail page, below or alongside existing content
       - Use existing tab pattern if the dashboard already has tabs, or add a new tabbed section at the bottom

    Style guidelines:
    - Diff view: use CSS custom properties for diff colors (e.g., --color-diff-add, --color-diff-remove) with fallbacks to green/red
    - JSON editor: dark background with monospace font matching existing code blocks
    - Responsive: stack panels vertically on mobile for the test console
  </action>

    cd frontend && npm run test:run (Level 1: Sanity — existing tests pass)
  </verify>
  <done>

  </done>
