  <files>backend/app/services/webhook_validation_service.py, backend/app/routes/webhook.py, backend/app/routes/github_webhook.py, backend/app/services/execution_service.py, backend/app/db/schema.py, backend/app/db/migrations.py, backend/tests/test_webhook_validation.py</files>
  <action>
    Based on webhooks.fyi HMAC Security best practices and existing codebase implementations:

    1. Create `backend/app/services/webhook_validation_service.py`:
       - Class `WebhookValidationService` with static/class methods

       - `validate_signature(raw_payload: bytes, signature_header: str, secret: str, algorithm: str = "sha256") -> bool`:
         - Support algorithms: sha256 (default), sha1 (legacy GitHub compat)
         - Parse signature format: "sha256=<hex>" or "sha1=<hex>" (detect from header prefix)
         - Compute HMAC with `hmac.new(secret.encode("utf-8"), raw_payload, hashlib.{algorithm}).hexdigest()`
         - Compare with `hmac.compare_digest()` (timing-safe, per research Don't Hand-Roll)
         - Return False if signature_header is None, empty, or has wrong prefix
         - This consolidates `ExecutionService._verify_webhook_hmac()` and `github_webhook.verify_github_signature()`

       - `validate_timestamp(timestamp_header: str, tolerance_seconds: int = 300) -> bool`:
         - Parse timestamp from header (Unix epoch or ISO 8601)
         - Reject if `abs(current_time - payload_time) > tolerance_seconds`
         - Return True if no timestamp header provided (optional per research -- not all providers include timestamps)
         - Default tolerance: 5 minutes (300 seconds)

       - `validate_webhook(request, secret: str, algorithm: str = "sha256", require_timestamp: bool = False, tolerance_seconds: int = 300) -> Tuple[bool, str]`:
         - Full validation pipeline: (1) get raw payload via `request.get_data()` BEFORE JSON parsing, (2) validate signature, (3) optionally validate timestamp
         - Return (True, "") on success, (False, "reason") on failure
         - Signature header names: `X-Hub-Signature-256` (GitHub), `X-Webhook-Signature` (generic), configurable

       - `validate_github(request, secret: str) -> Tuple[bool, str]`:
         - GitHub-specific: uses `X-Hub-Signature-256` header, sha256 algorithm
         - Convenience wrapper around `validate_webhook()`

    2. Update `backend/app/routes/webhook.py`:
       - Replace inline HMAC validation with `WebhookValidationService.validate_webhook()`
       - When validation fails: return 403 with `{"error": "Invalid webhook signature"}` or `{"error": "Webhook replay detected"}`
       - Validation happens BEFORE JSON parsing (existing code already does `request.get_data()` first -- preserve this)

    3. Update `backend/app/routes/github_webhook.py`:
       - Replace `verify_github_signature()` function with `WebhookValidationService.validate_github()`
       - Remove the standalone `verify_github_signature()` function
       - Return 403 with descriptive error on validation failure

    4. Remove `ExecutionService._verify_webhook_hmac()` static method:
       - All callers should use `WebhookValidationService.validate_signature()` instead
       - Update any remaining references in execution_service.py

    5. Add replay protection DB table (optional, for tracking seen timestamps):
       - Table `webhook_replay_log` with columns: `payload_hash TEXT`, `timestamp TEXT`, `created_at TEXT`
       - This is lightweight -- used to detect replayed payloads within the tolerance window
       - Or: rely purely on timestamp-based validation without DB tracking (simpler, sufficient for most cases)
       - Decision: use timestamp-only validation (no DB table) -- simpler, avoids additional write per webhook, the tolerance window is short enough (5 min) that replays outside the window are already rejected

    6. Write tests in `backend/tests/test_webhook_validation.py`:
       - Test valid HMAC signature returns True
       - Test invalid HMAC signature returns False
       - Test missing signature header returns False
       - Test wrong algorithm prefix returns False
       - Test sha1 algorithm (legacy) works
       - Test timestamp validation: within tolerance -> True
       - Test timestamp validation: outside tolerance -> False
       - Test timestamp validation: no header + not required -> True
       - Test full validate_webhook pipeline with Flask test request
       - Test 403 response on invalid signature via route test
       - Test valid signature passes through to trigger dispatch
       - Use `isolated_db` fixture and Flask test client
