  <name>Task 1: Implement pause/resume in ProcessManager and execution status tracking</name>
  <files>backend/app/services/process_manager.py, backend/app/services/execution_log_service.py, backend/app/db/triggers.py</files>
  <action>
    Based on Python subprocess SIGSTOP/SIGCONT documentation and existing ProcessManager.cancel_graceful() pattern:

    1. Add `pause()` and `resume()` classmethods to `ProcessManager`:
       - `pause(execution_id: str) -> bool`:
         - Acquire lock, get ProcessInfo from `_processes` dict
         - Call `update_execution_status_cas(execution_id, "paused", expected_status="running")` -- only pause if currently running (prevents race with completion per research Pitfall 5)
         - If CAS succeeds: `os.killpg(info.pgid, signal.SIGSTOP)` to pause entire process group
         - Track pause timestamp in ProcessInfo (add `paused_at: Optional[float]` field to ProcessInfo dataclass)
         - Schedule auto-cancel timer: `threading.Timer(PAUSE_TIMEOUT, cls._auto_cancel_paused, args=[execution_id])` with `daemon=True`
         - Store timer reference in ProcessInfo for cleanup
         - Return True on success, False if process not found or CAS fails
       - `resume(execution_id: str) -> bool`:
         - Acquire lock, get ProcessInfo
         - Call `update_execution_status_cas(execution_id, "running", expected_status="paused")` -- only resume if currently paused
         - If CAS succeeds: `os.killpg(info.pgid, signal.SIGCONT)`
         - Cancel the auto-cancel timer if it exists
         - Clear `paused_at` timestamp
         - Return True on success
       - `_auto_cancel_paused(execution_id: str)`:
         - Called by timer after PAUSE_TIMEOUT (30 minutes = 1800 seconds)
         - Check if still paused (CAS with expected_status="paused")
         - If still paused: send SIGCONT first (required before SIGTERM on stopped process), then call existing `cancel_graceful()`
         - Update status to "pause_timeout" via `update_execution_log()`

    2. Add "paused" and "pause_timeout" as valid execution states:
       - Update `ExecutionState` class in execution_service.py to include PAUSED = "paused" and PAUSE_TIMEOUT = "pause_timeout"
       - These are non-terminal states (paused) and terminal states (pause_timeout)

    3. Update `ProcessInfo` dataclass:
       - Add `paused_at: Optional[float] = None`
       - Add `pause_timer: Optional[threading.Timer] = None`

    4. Update `cleanup()` to cancel pause timer if it exists

    5. SSE streaming behavior during pause:
       - The subprocess stdout/stderr pipes will naturally block when the process is stopped (SIGSTOP)
       - The existing `_stream_pipe` threading in ExecutionService will simply wait (blocked read) -- no changes needed for log streaming
       - Add a "paused" status event to the SSE stream via `ExecutionLogService` so the frontend knows the execution is paused

    Avoid: Sending SIGTERM to a stopped process without SIGCONT first (it won't be received). Always SIGCONT before SIGTERM.
