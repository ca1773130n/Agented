<task type="auto">
  <name>Task 1: Create useEventSource composable</name>
  <files>frontend/src/composables/useEventSource.ts</files>
  <action>
    Create the shared SSE composable that wraps `createAuthenticatedEventSource` from `../services/api/client`.

    **Design principles (from research):**
    - The composable handles ONLY connection lifecycle: connect, disconnect, status tracking, event registration, unmount cleanup
    - Event-specific parsing stays in each consumer composable (state_delta seq tracking in useAiChat, content/timestamp dedup in useConversation, output/complete/error in useProjectSession)
    - The composable does NOT dictate message formats or parsing logic

    **Implementation:**

    1. Define types:
       ```typescript
       type SSEStatus = 'idle' | 'connecting' | 'open' | 'error' | 'closed';

       interface UseEventSourceOptions {
         url: string | (() => string);
         events?: Record<string, (event: MessageEvent) => void>;
         onOpen?: () => void;
         onError?: (event: Event) => void;
         onMessage?: (event: MessageEvent) => void;
         autoConnect?: boolean; // default: false
       }

       interface UseEventSourceReturn {
         status: Ref<SSEStatus>;
         connect: () => void;
         close: () => void;
         /** Access the underlying source for advanced use (e.g., adding/removing listeners dynamically) */
         getSource: () => AuthenticatedEventSource | null;
       }
       ```

    2. Implement `useEventSource(options: UseEventSourceOptions): UseEventSourceReturn`:
       - Create `status = ref<SSEStatus>('idle')`
       - Track the source instance via `let source: AuthenticatedEventSource | null = null`
       - `connect()` function:
         - Call `close()` first to clean up any existing connection
         - Set `status.value = 'connecting'`
         - Resolve URL: if `options.url` is a function, call it; otherwise use directly
         - Call `createAuthenticatedEventSource(resolvedUrl)` to create the source
         - Set `source.onopen = () => { status.value = 'open'; options.onOpen?.(); }`
         - Set `source.onerror = (event) => { status.value = 'error'; options.onError?.(event); }`
         - If `options.onMessage`: set `source.onmessage = options.onMessage`
         - If `options.events`: iterate entries and call `source.addEventListener(eventName, handler)` for each
       - `close()` function:
         - If `source` exists: call `source.close()`, set `source = null`
         - Set `status.value = 'closed'`
       - `getSource()`: returns current `source` reference
       - If `options.autoConnect !== false` is NOT the case -- default is false (consumers call connect() explicitly when they have the URL ready)
       - If `options.autoConnect === true`: call `connect()` immediately
       - Register `onUnmounted(close)` for automatic cleanup
       - Return `{ status, connect, close, getSource }`

    3. Export the composable as a named export: `export function useEventSource(...)`
    4. Also export the types: `export type { SSEStatus, UseEventSourceOptions, UseEventSourceReturn }`

    **Key design decisions:**
    - `autoConnect` defaults to `false` because all three consumers need to set up event handlers before connecting, and the URL is often dynamic (depends on session/conversation ID)
    - URL as `string | (() => string)` handles both static and dynamic URLs (useAiChat builds URL from reactive sessionId)
    - `getSource()` is a function (not reactive) because consumers need the raw EventSource for advanced operations (e.g., dynamically adding listeners after connect)
    - `events` map is set at connect time -- for consumers that need to change handlers, they should close and reconnect
