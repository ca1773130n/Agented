    **CON-09: Shared useAsyncState composable**

    1. Create `frontend/src/composables/useAsyncState.ts` with a generic composable that standardizes the loading/error/data lifecycle:

       ```typescript
       import { ref, type Ref } from 'vue';

       export interface UseAsyncStateReturn<T> {
         data: Ref<T>;
         isLoading: Ref<boolean>;
         error: Ref<string | null>;
         execute: () => Promise<void>;
       }

       export function useAsyncState<T>(
         asyncFn: () => Promise<T>,
         initialValue: T
       ): UseAsyncStateReturn<T> {
         const data = ref<T>(initialValue) as Ref<T>;
         const isLoading = ref(false);
         const error = ref<string | null>(null);

         async function execute(): Promise<void> {
           isLoading.value = true;
           error.value = null;
           try {
             data.value = await asyncFn();
           } catch (err: unknown) {
             error.value = err instanceof Error ? err.message : String(err);
           } finally {
             isLoading.value = false;
           }
         }

         return { data, isLoading, error, execute };
       }
       ```

    2. Export `useAsyncState` from the composables barrel file if one exists, or ensure it is importable from the composables directory.

    3. Do NOT refactor existing composables (useDataPage, etc.) to use useAsyncState in this plan. The composable is created as infrastructure for future adoption and for new code. Refactoring existing composables would cascade changes through too many consumer components and is out of scope for this plan.
